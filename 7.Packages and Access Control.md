# 7.1 Packages
## 包的意义

- 创建与使用不同项目中同名的类时，在访问或使用它们时减少歧义，调用正确的类。

> 本质是一个用于组织类和接口的**命名空间**。

## 命名规范

- 使用域名反转实现包名的唯一性（Josh Hug 的个人网站是 `joshh.ug`，其动物类库的包名应定义为 `ug.joshh.animal`）
- 个人demo可以不写，但若库要传递必须要遵守

## 访问写法推荐

| 使用场景        | 推荐引用方式        | 架构选择理由       |
| ----------- | ------------- | ------------ |
| **同包内类访问**  | **简单名称**      | 代码最简洁，易读性最高。 |
| **跨包频繁访问**  | **导入 **       | 顶层声明减少冗余     |
| **同名类冲突处理** | **规范名称 **     | 使用全限定名以消除歧义。 |
| **开发建议**    | **显式 import** | 防潜在的命名遮蔽风险。  |

## 包创建与储存规则

- 将包名放在该包的每个文件顶部
```java
package ug.joshh.animal;

```

- 将文件存储在具有相应文件夹名称的文件夹中。文件夹应该有一个与包裹相符的名称
```
`ug.joshh.animal` 包裹在 ug/joshh/animal 文件夹中
```

> 确保了编译器能够通过确定的算法快速定位源代码


### IDEA中包的创建

- **新建包：** `File -> New -> Package`。**专家提示：** 在输入名称时使用点号（如 `ug.joshh.animal`）， 会一次性自动创建所有嵌套的子目录。
- **添加新类：** 右键点击包名选择 `New -> Java Class`，IDE 会自动填充 `package` 声明并将其存入正确目录。
- **迁移旧文件：** 若要将现有类移入包内，需先在文件顶部添加 `package` 声明，按报错上下文提示迁移即可

## 默认包

- 任何没有文件顶部显式包名的 Java 类自动被视为“默认”包的一部分.
- 写真实程序时，应避免
- 无法导入其他包,同时容易与其他package private类发生同名

> Java 文件通常应该以一个显式的包声明开头.


## JAR

- JAR 是一种基于 ZIP 格式的压缩包，包含了类文件、资源.(许多 `class` 文件会导致分发效率低下)

### IDEA中JAR创建

1. 进入 `File -> Project Structure -> Artifacts`。
2. 点击 `+` 号，选择 `JAR -> From modules with dependencies`。
3. 在弹出窗口中确认主类（Main Class）及 Manifest 存储位置，点击 OK。
4. 执行顶部菜单 `Build -> Build Artifacts`，在弹出的悬浮窗中点击 `Build`。生成的 JAR 将出现在工程目录的 `out/artifacts` 下。


# 7.2 Access Control

- 访问控制是为了限制代码的可见性,进而实现封装性

> 保护内部状态不被外部篡改
> 保留了在不破坏外部调用逻辑的前提下，对内部实现可重构


## 访问权限对比

| 修饰符 (Modifier)          | 本类  | 同包  | 子类(非同包) | 外部  |
| ----------------------- | --- | --- | ------- | --- |
| **public**              | Yes | Yes | Yes     | Yes |
| **protected**           | Yes | Yes | Yes     | No  |
| **package-private(默认)** | Yes | Yes | No      | No  |
| **private**             | Yes | No  | No      | No  |
> 对于接口，其方法的默认访问权限实际上是公开的，而非包私有的
> 
## 四大权限

- 私有权限 ：本类中的私有实现,防任何干预
- 包级私有 ：允许包内类访问逻辑,实现模块协作
- 受保护权限 ：子类使用父类方法与逻辑,用者不不准干预
- 公共权限 ：实现接口,允许任何人使用

 >**所有 package-private 的成员在默认包内是完全互通的.**

> **访问权限仅基于变量的静态类型进行判定**
