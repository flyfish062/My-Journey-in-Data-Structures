- `Iterable`接口，它使一个类能够被迭代，并且需要 `__init__` 方法`iterator()`，该方法返回一个 `Iterator` 对象。我们也看到了`Iterator`接口，它定义了一个对象，并提供了执行迭代的方法。你可以把 `Iterator` 想象成一个机器，它被放置在可迭代对象上，帮助执行迭代操作。任何可迭代对象都是 `Iterator` 执行迭代操作的对象。



- `next()`：它返回列表的这一个元素，并在此处将其打印出来。同时，它还将迭代器向前移动一位。

- `hasnext()`：判定这一位有无可用元素。

## 重写`toString()`

- `toString()`方法返回对象的字符串表示形式。该`System.out.println()`函数会隐式地对传递给它的任何对象调用此方法，并打印返回的字符串。

```java
//正常写法
Dog dog  = new Dog(...);
System.out.println(dog);

//底层调用
String s = dog.toString();
System.out.println(s);

```

> 默认`Object`类的 `toString()`print 方法会打印对象在内存中的位置，返回一个十六进制字符串,
> 返回`类名@0x...`形式字样




**重写`toString`以实现{1, 2, 3, 4}形式的输出**
```java
public String toString() {
    String returnString = "{";
    for (int i = 0; i < size; i += 1) {
        returnString += myArray[i];
        returnString += ", ";
    }
    returnString += "}";
    return returnString;
}
```


- 这个解决方案不够完美。在 Java 中使用字符串拼接时，`returnString += myArray[i];`并非是在字符串末尾添加内容`returnString`，而是创建了一个全新的字符串。这效率极低，因为创建一个新的字符串对象也需要时间。

> 时间复杂度与字符串长度呈线性关系。O(n ^ 2)

# 改进


 > Java 有一个名为 `String` 的特殊类`StringBuilder`。它创建一个可变的字符串对象，因此可以继续向同一个字符串对象添加内容，而不是每次都创建一个新的对象。


```java
public String toString() {
//1. 处理空集情况
    if (size == 0) {
     return "{}"; 
     }
        StringBuilder returnSB = new StringBuilder("{");
        for (int i = 0; i < size - 1; i += 1) {
            returnSB.append(items[i].toString());
            returnSB.append(", ");
        }
        returnSB.append(items[size - 1]);
        returnSB.append("}");
        return returnSB.toString();
    }
```





## 重写`equals()`

> `equals()`在 Java 中，它们`==`的行为有所不同。`==`检查内存中的两个对象是否实际上是同一个对象。记住，要按值传递！`==`检查两个容器是否存放着相同的内容。对于基本类型，这意味着检查值是否相等。对于对象，这意味着检查地址/指针是否相等


 - `equals(Object o)``this` 是对象中的一个方法，默认情况下它的作用类似于 `==`，即检查 `this` 的内存地址是否与 `o` 相同。**重写它，即出现元素相同，顺序不同，也认为在内容上看是同一个set容器**


```java
public boolean equals(Object other) {
        if (this == other) {
            return true;
        }
        if (other == null) {
            return false;
        }
        
        if (other.getClass() != this.getClass()) {
            return false;
        }
        ArraySet<T> o = (ArraySet<T>) other;
        if (o.size() != this.size()) {
            return false;
        }
        for (T item : this) {
            if (!o.contains(item)) {
                return false;
            }
        }
        return true;
    }
    
    
//确保我们的等号能够处理空值和不同类的对象。此外，我们还优化了函数，如果 == 方法返回 true，
//就直接返回 true。这样，我们就避免了遍历集合的额外工作
```
