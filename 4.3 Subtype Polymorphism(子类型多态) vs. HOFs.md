
- **多态性**指的是对象可以有多种形式或类型。在面向对象编程中，多态性涉及对象如何被视为其自身类的实例、其上类的实例...

## 多态性
- 同一接口，多种实现

#### A. 子类型多态 (`Subtype Polymorphism`) —— **最常见的“多态”**

- **定义：** 一个变量可以指向其子类的对象。程序在运行时决定具体调用哪个类的方法。
    
- **Java体现：** 继承（extends）和接口实现（implements）。
    
- **绑定时间：** **运行时（Runtime）**，也叫动态绑定。
    
```java
    Animal dog = new Dog(); // 子类型 Dog 赋值给父类型 Animal
    dog.makeSound();       // 运行时才确定调用 Dog 的 makeSound
```   

#### B. 参数化多态 (`Parametric Polymorphism`)

- **定义：** 代码逻辑与具体的类型无关，通过类型参数来处理多种数据类型。
    
- **Java体现：** **泛型（Generics）**。
    
- **绑定时间：** **编译时（Compile-time）**，但通过“类型擦除”实现。
    
```java
    List<String> list = new ArrayList<>(); // List 可以处理任何对象类型
```

#### C. 特定多态 (`Ad-hoc Polymorphism`)

- **定义：** 不同的类型使用相同的函数名，但逻辑完全不同。
    
- **Java体现：** **方法重载（Overloading）**。
    
- **绑定时间：** **编译时（Compile-time）**，也叫静态绑定。
    
- **例子：**
    
    codeJava
    
    ```
    print(int i);
    print(String s); // 同名函数，参数不同
    ```
    

#### D. 强制多态 (`Coercion Polymorphism`)

- **定义：** 编译器隐式地将一种类型转换为另一种类型，以匹配操作。
    
- **Java体现：** **隐式类型转换**。

```java
    double x = 1; // 将 int 1 强制转换为 double 1.0
```

## 显式高阶函数方法 (`Higher-Order Function`)

- 对象是固定,不被捆绑的，逻辑是通用的。提供一个通用的“比较框架”，然后注入具体的通用的“比较规则”

- **逻辑所在：** 逻辑不在对象内部，而在调用处或独立的策略类中。
    
- **谁做决定：** **调用者**通过注入 Lambda 表达式来做决定。
    
- **灵活性：** 极高。可以根据心情决定是按长度比、按字典序比、还是按元音字母数量比。


## 子类型多态性方法 (`Subtype Polymorphism`)
- 类型与逻辑是绑定的。类型本身定义了“该如何比”，只需要调用一个统一的接口

- **逻辑所在：** 逻辑封装在**类（Class）**的内部。
    
- **谁做决定：** **对象本身**通过其所属的子类实现来做决定。
    
- **灵活性：** 较低。一个类通常只有一种“自然排序”或核心比较行为。


## Generalization(通用化)

在使用子类型多态性方法时,对于多个类均实现比较功能,会出现
- **代码冗余 ：** 每个函数的逻辑几乎一模一样。

- **维护困难：** 每次增加一个新物种，都要复制粘贴一遍代码。

- **不支持运算符重载:**   > 运算符不适用于自定义对象

- **Object 类型的抽象性：** Object 是所有类的父类。编译器不知道如何比较两个 Object

## 接口与类型

> 接口不仅是行为的集合，更是**类型的保证**。编译器利用接口进行严格的类型检查。

- 如果 类声明了接口但没重写接口中的方法，编译器会报错。

- 如果 类没声明接口，但在函数中里把类传给接收 接口类 的函数，编译器报错

## 弊端

> 自定义的子类型多态性方法的通用化仍有缺陷

- **强制转型：** 必须在类中重写接口方法,将Object类转化为实现类时，这既麻烦又容易在运行时出错。
 - **孤立性：** Java 标准库不认识你的 `OurComparable`接口(string,int,double等类不能实现转换

> 从手动自定义创建接口转变为使用标准的泛型接口是解决方法


## 泛型接口

- **类型安全：** `compareTo(Dog uddaDog)` 直接接收 Dog 类型，**消除了手动强转**。
- **生态融合：** 实现标准接口后，对象可以立即被 Java 所有的内置算法库处理。

> 泛型接口虽然实现了在实际类对接口类方法的重写中的强制转换,适配Java原生类与库,但没解决使用完接口后想使用实际类时所需的类型强制转换

- Comparable是Java 官方自带的接口,可以直接使用--用于泛型
```java
public class Dog implements Comparable<Dog> {
    ...
    public int compareTo(Dog uddaDog) {
        return this.size - uddaDog.size;
    }
}
```

```java
public interface Comparable<T>{
   public int compareTo(T obj);
}
```

**通过泛型编程,自定义子类型多态性方法**

- 将综合函数抽象出了 一个"能力"(指不受类型影响的逻辑方法)函数,和一个根据类型可以实现子类型多态的接口函数

- 解决了多个实现继承类的重写方法的代码繁杂,提高了复用性

- 同时解决了类重写接口函数时要解决的强制转换问题

> 目前一切都是在**让一个算法适用于不同的类**,而更可以实现**让同一个类适用于不同的比较场景**


## ## 自然排序 vs. 定制排序 (`Comparable` vs. `Comparator`)

| 特性       | **Comparable **                               | **Comparator**               |
| -------- | --------------------------------------------- | ---------------------------- |
| **定义位置** | 嵌入在类内部                                        | 通常作为独立的策略（如嵌套类或独立类）          |
| **核心方法** | compareTo(T o)                                | compare(T o1, T o2)          |
| **排序类型** | **自然排序 (Natural Order)**：通常只有一种（如 Dog 按 size） | **定制排序**：可以有无数种（按名字、按品种、按年龄） |
| **灵活性**  | 较低，一旦写死很难更改                                   | 极高，可以动态更换比较规则                |

## `Comparator`

- 让同一个类适用于不同的比较场景

- **`Comparator` 是 Java 官方自带的、标准库中的接口。位于 java.util** 包中。
```java
import java.util.Comparator;

public class Dog implements Comparable<Dog> {
    ...
    public int compareTo(Dog uddaDog) {
        return this.size - uddaDog.size;
    }

    private static class NameComparator implements Comparator<Dog> {
        public int compare(Dog a, Dog b) {
            return a.name.compareTo(b.name);
        }
    }

    public static Comparator<Dog> getNameComparator() {
        return new NameComparator();
    }
}
```


- 可以定义无数个 `Comparator`（按名字、按年龄、按品种），在需要的时候随取随用

# 接口即“回调”

**什么是回调？**  
- 当写函数时，不知道未来会有什么新的对象或者新的比较规则。函数空有一套流程，却缺少一个的因类型或者规则改变的核心逻辑环节。

**Java 的解决方案：**  

- 把函数包装在接口里。
- 函数调用接口的那一刻，这种“回头调用”的行为就叫回调。

**子类型多态的体现：**  

- max 只需要持有一个 `Comparator` 的接口引用。在运行时，你传给它 `NameComparator`，它就按名字比；你传给它 `SizeComparator`，它就按大小比。**同一行代码，展现出完全不同行为。**
