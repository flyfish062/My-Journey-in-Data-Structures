
# `Compiler`(编译器)/`interpreter`(解释器)

```Bash
javac HelloWorld.java
# javac 调用java complier，将HelloWorld,java转化为HelloWorld.class
java HelloWorld
# 调用interpreter -> stuff happens
#output
Hello World!
```

## 编译的注意细节

> 不遵守下列规则，均导致无法编译，更不可使用解释器

- 最外层类名要和文件名一样
- 保证**静态类型检查**

# Static Typing（静态类型）

当一个Java的变量被定义，他的类型不可更改

- 不存在类型错误（py)
- 变量，函数等类型确定，容易调试，理解

## 一些不同于C++的写法

- Java所有代码均要在类中
- Java最外层必须定义一个 public/private class name,类似身份标识，必须和文件名等同
- Java定义函数时均标出public/private 
- 注释遵循 `Javadoc`格式,通常包含描述标签
```java
/** this is a commnet */
```
- 实例化时采用`new`关键字
## Two method: client/non-client

- Client : 类（包含方法和成员）与类的特定功能调用**分离**
- non-client : 类与类的调用合一

# public static void main(String[] args)

```java
public class ArgsDemo{
  /** Prints out the 0th command line argument. */
  public static void main(String[] args){
    System.out.println(args[0]);
  }
}
```
```bash
java ArgsDemo hello world
hello
```
