
# Extends Inheritance(延伸继承)
- A类要继承B类的方法，重用已定义类的代码
```java
public class RotatingSLList<Item> extends SLList<Item>
```

> `extends` 关键字保留了 `SLList` 的原始功能，同时允许进行修改和添加额外功能。

- 通过使用`extends`关键字，子类继承父类的所有成员
**成员包括**
- 所有实例变量和静态变量
- 所有方法
- 所有嵌套类

> 注意构造函数不能继承，私有成员不能被子类直接访问。

# Constructors Are Not Inherited(super)

> 虽然构造函数不继承，但 Java 要求所有构造函数必须以调用其超类构造函数之一

```java
public class Human {...}
public class TA extends Human {...}

TA Christine = new TA();//调用了超类的构造函数
```


## super

- 解决子类构造函数问题
- 解决子类重写父类方法问题（访问父类被隐藏的成员）

> 确保了**代码的复用性**（调用父类方法）
> **初始化的安全性**（调用父类构造）
> 同时也解决了**命名冲突**的问题。
### super用法

#### 调用父类的构造方法

子类的构造方法内部可以通过 super() 调用父类的构造方法。

- **规则**：必须放在子类构造方法的第一行。
    
- **默认行为**：如果你不写 super()，Java编译器会自动在子类构造函数的第一行插入一个隐式的 super()（**调用父类的无参构造**）。

#### 访问父类中被重写（Override）的方法

- 子类重写了父类的方法，但又想在子类方法内部复用父类的逻辑时,使用 `super.methodName()`。

```java
class Parent {
    void message() {
        System.out.println("这是父类消息");
    }
}

class Child extends Parent {
    @Override
    void message() {
        super.message(); // 调用父类的方法
        System.out.println("这是子类消息");
    }
}
```

#### 访问父类中被隐藏（Shadowing）的成员变量(重写成员变量的方法）

- 如果子类和父类定义了同名的变量，可以使用 `super.variableName` 来明确指向父类的变量

```java
class Parent {
    int age = 50;
}

class Child extends Parent {
    int age = 20;

    void display() {
        System.out.println("子类 age: " + this.age);  // 20
        System.out.println("父类 age: " + super.age); // 50
    }
}
```

#### 注意事项

-  **不能越级**：super 只能指向**直接父类**。不能通过 super.super 来访问“爷爷类”的成员。

-  **静态限制**：由于 super 指向的是一个具体的父类对象实例，因此它不能在静态（static）方法或静态代码块中使用。

- **构造冲突**：在同一个构造方法中，this() 和 super() 不能同时出现，因为都要求放在第一行。

# The Object Class(对象类)

![类层次图](object%20class.png)

> Java 中的每个类都是对象类的后代，或 `extends` 对象类。即使是类中没有显式`extends`的类，也仍然隐式地 `extends` 了对象类。

- 对象类提供了每个对象都应能执行的——如`.equals(Object obj)`、`.hashCode()`和`toString()`。


> `extends` 关键字定义了“is-a”，即`hypernymic`(上位词)关系。**不要将其用于“has-a”，即`meronynic`(部分)关系**

# Encapsulation（封装）

- 面向对象编程的基本原则之一
- 面对编程复杂性的采取方法


模块可以定义为一组方法，这些方法整体协同工作以执行某项任务或相关任务。这可以是一个代表列表的类。如果模块的实现细节内部隐藏，且唯一通过文档化接口进行交互，则该模块称为封装。
## 理念

- 程序应该被构建成模块化、可互换的部分，可以互换而不破坏系统
- 层级抽象

## Abstraction Barriers（抽象障碍）and  How Inheritance Breaks Encapsulation
- 用户不应能够观察他们所使用某个数据结构的内部运作
- 在 Java 中使用 `private` 关键字,确保底层复杂性不会暴露给外部世界


- 继承破坏了封装性，主要是因为继承引入了**类与类之间的强耦合**，使得子类依赖于父类的内部实现细节

- 封装的本质是“隐藏实现细节，只暴露接口”。但在继承关系中，子类往往需要了解父类的内部实现才能正确地重写方法或扩展功能。

- 如果父类的内部实现发生了变化，即使父类的对外接口（API）没变，子类也可能因此崩溃。

封装性的破坏：**子类必须知道父类的方法是如何相互调用的**，才能保证逻辑正确


## Type Checking and Casting(类型检查和类型转换)

- **动态方法查找**是根据对象的动态类型确定运行时执行的方法的过程。具体来说，如果父类中的某个方法被子类覆盖，那么运行时调用的方法由该变量的运行时类型或动态类型决定。

```java
VengefulSLList<Integer> vsl = new VengefulSLList<Integer>(9);
SLList<Integer> sl = vsl;
// sl 静态类型为SLList,动态为VengefulSLList

sl.addLast(50);//调用SLList,因为VengefulSLList没有重写addLast方法
sl.removeLast();//调用VengefulSLList，VengefulSLList重写了这个方法，动态方法查找生效

/*会报错**/sl.printLostItems();//VengefulSLList的新方法，但不是重写父类，动态查找不生效，只会在SLList中找方法，找不到，故报错

/*会报错**/VengefulSLList<Integer> vsl2 = sl;
//编译器只允许基于编译时类型进行方法调用和赋值。由于编译器只识别静态类型`sl`是SLList，因此不允许VengefulSLList的“容器”存储该类型。
```
