### 算法

#### 算法基本概念
对特定问题的求解方法与步骤的一种描述。一个被明确定义的、计算机可执行的有限步骤序列，一个指令可表示多个操作。
**描述方式**
- 自然语言
- 流程图
- 伪代码/类语言
- 程序代码

#### 算法特性

1. **有穷性（`Finiteness`）**  
    算法必须在执行有限个步骤后自动终止，不会无限循环，且每一步都可在有限时间内完成。
    
2. **确定性（`Definiteness`）**  
    算法中的每一步操作都有确切的含义，不存在二义性，在任何条件下都能得到唯一确定的结果。
    
3. **可行性（`Effectiveness`）**  
    算法中的每一步都必须是可行的，即能够通过已经实现的基本运算在有限时间内完成。
    
4. **输入（`Input`）**  
    算法有零个或多个输入。
    
5. **输出（`Output`）**  
    算法至少产生一个输出。

#### 算法设计的要求

1. **正确性（`Correctness`）**  
    算法必须对所有合法输入都能产生符合问题规格的正确输出，能够处理边界条件和特殊情况，逻辑严密无误。要求对精心选择的数据集能正确输出结果
2. **可读性（`Readability`）**  
    算法描述应结构清晰、逻辑层次分明、命名规范、注释充分，便于他人理解、验证、调试和维护。
3. **健壮性（`Robustness`）**  
    算法能够识别并合理处理非法或异常输入（如空值、越界、格式错误等），不因异常情况导致系统崩溃，并提供适当的错误反馈或容错机制。
4. **高效性（`Efficiency`）**  
    算法应在时间和空间资源上表现优良：时间复杂度低（执行速度快），空间复杂度低（占用内存少），在实际应用中具备良好的性能表现。

### 算法分析

满足上述要求后，考虑

- **时间效率**：算法耗费的时间
- **空间效率**：算法执行过程中耗费的储存空间

####  算法时间效率的度量

用依据该算法编制的程序在计算机上执行所消耗的时间来度量。

**事后统计**
    - 将算法实现，测算其时间和空间开销。
        - 缺点：编写程序实现算法将花费较多的时间和精力；所得实验结果依赖于计算机的软硬件等环境因素，掩盖算法本身的优劣
**事前分析**
    - 对算法所消耗资源的一种估算方法。
        - 方法：计算机执行一次操作所需的时间（与机器有关）（单位时间）与算法中进行操作次数的乘积



#### 时间复杂度的渐进表示法

为了便于比较不同算法的时间效率，我们仅比较它们的数量级  

若有某个辅助函数 f(n)，使得当 n 趋近于无穷大时，T(n)/f(n) 的极限值为**不等于零的常数**，则称 f(n) 是 T(n) 的同数量级函数。

记作 T(n)=O(f(n)))，称 O(f(n)) 为算法的**渐进时间复杂度**（O是数量级的符号），简称**时间复杂度**。

算法中**基本语句重复执行的次数**是问题规模n的某个函数f(n)，算法的时间量度记作： **T(n)=O(f(n))**
它表示随着n的增大，算法执行的时间的增长率和f(n)的增长率相同，称**渐近时间复杂度**。

#### 时间复杂度

若有某个辅助函数 f(n)，使得当 n 趋近于无穷大时，T(n)/f(n)的极限值为**不等于零的常数**，则称 f(n)f是 T(n)的同数量级函数。记作：  
**T(n)=O(f(n))**

 称 **O(f(n))为算法的**渐进时间复杂度**（O是数量级的符号），简称**时间复杂度**。
 
#### 算法时间复杂度

- **最坏时间复杂度**：指在**最坏情况**下，算法的时间复杂度。
- **平均时间复杂度**：指在**所有可能输入实例**在**等概率出现**的情况下，算法的期望运行时间。
- 一般总是考虑在**最坏情况**下的时间复杂度，以保证算法的运行时间不会比它更长。

#### 空间复杂度

- **空间复杂度**：算法所需存储空间的度量，  
    记作： **S(n)=O(f(n))  
    其中 n 为问题的规模（或大小）

**算法要占据的空间**
- 算法本身要占据的空间，输入/输出，指令，常数，变量等
- 算法要使用的**辅助空间**
